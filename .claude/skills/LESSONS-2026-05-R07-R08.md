## R07 — sitemap 假阳性修复

**Bug**: bandcamp/phet/wolfram/imgur 5+ 站 `/sitemap.xml` 返 200 HTML 错误页（非 XML sitemap），`sub_sitemaps_fetched=5 / urls_total=0` 看起来 found，下游搞混"有 sitemap"vs"5 个候选全垃圾"。

**Patch**: `extract_sitemap.py` curl_bytes 后检查 body 前 4KB 必须含 `<urlset` 或 `<sitemapindex`，否则标 `code=-1`（not-a-sitemap），不计入 `sub_valid`。新加 `sub_sitemaps_valid` 字段到 `_sitemap_index.json`。

**验证**:
- bandcamp: fetched=5 / **valid=0** / urls=0
- phet: fetched=5 / **valid=0** / urls=0
- wolfram: fetched=5 / **valid=0** / urls=0
- imgur: fetched=5 / **valid=0** / urls=0
- 真有 sitemap 站（如 booking）valid 与 fetched 一致

跨站影响：所有"假阳性"站现在能被 audit 正确归类为"无 sitemap"，design-tasks 不会被误导。

## R08 — JSON.parse("...") escape 包装 SSR

**Bug**: chess.com / Redux / Apollo 类站用 `JSON.parse("...escaped JSON...")` ship dehydrated state，不在我们之前的 9 个 STATE pattern 里。chess 报告 "inline 多处 JSON.parse"。

**Patch**: `extract_server_state.py` 加 `JSON_PARSE_RE` pattern；扫所有 `JSON.parse("...")` blob ≥ 100 字符，unicode_escape 解后存 `json_parse_blobs[]` to server_state.json。

**验证**:
- 全 96 站 SSR: 93,534 → **94,579 KB**（+1 MB JSON.parse blobs）
- chess 单站仍 179 KB（brand_namespace_bootstrap），无 JSON.parse hit — chess 用的是 brand-namespace 不是 JSON.parse；其他 Redux/Apollo 类站才命中
