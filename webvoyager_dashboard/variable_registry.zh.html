Variable Registry

WebHarbor 推荐变量注册表

这个页面不是把所有 DB 字段机械列出来，而是按当前 WebHarbor 实际 UI 和 seed DB 覆盖，挑出适合排列组合造题的变量。 组合前仍然要先查 DB 和跑浏览器验证。

先用这些站点做批量生成

Google Flights

变量空间最大，航班表 126k+ 行。优先组合 origin、destination、depart、return、cabin、stops、price、airline、emissions、duration。

Booking

酒店题最容易参数化。优先组合 city、dates、adults/rooms、rating、stars、price、amenities、brand、distance。

Amazon

电商筛选和状态流丰富。优先组合 category、query、brand、price、rating、reviews、condition、variant、shipping/returns。

Google Map

适合空间推理。优先组合 city/zip/anchor、category、rating、hours、distance、route mode、review field。

ESPN

表格题很稳。优先组合 sport、date、team、conference、stat、position、schedule、standings、injury。

组合规则

先查 DB

每个变量组合必须先返回至少一个答案实体。0 结果直接丢弃。

再查页面

答案字段必须能在搜索结果页、详情页或确认页上看见。

限制组合阶数

第一阶段只做 2 到 4 个变量的组合。高阶组合容易 0 结果，也难评估。

保留干扰项

好题不应该只有一个结果，但目标答案要能通过约束稳定选出。
