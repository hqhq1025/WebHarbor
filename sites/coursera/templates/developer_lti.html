Learning Tools Interoperability (LTI) Integration

Embed Coursera courses inside Canvas, Blackboard, Moodle, D2L and other LMS platforms using the IMS Global LTI {{ lti_version }} Advantage standard. This page documents the OIDC handshake, deep linking, and grade passback flow.

Quick facts

  • LTI version: {{ lti_version }} (Advantage)
  • OIDC callback URL: {{ oidc_callback_url }}
  • Tool launch URL: {{ launch_url }}
  • Deep linking endpoint: {{ deep_link_url }}
  • Public JWKS endpoint: {{ keyset_url }}
  • Issuer (iss): https://coursera-mirror.local
  • Supported message types: LtiResourceLinkRequest, LtiDeepLinkingRequest

Step 1 — Register the tool

In your LMS admin console, register Coursera as an external tool with the values above. Canvas calls this "Developer Keys", Blackboard calls it "Manage LTI Tool Providers".

Step 2 — OIDC handshake

The LMS will POST to the OIDC callback URL with iss, login_hint, and target_link_uri. Coursera responds with a JWT-shaped resource link payload.

$ curl -X POST {{ request.host_url.rstrip('/') }}{{ oidc_callback_url }} \
    -d "iss=https://canvas.instructure.com" \
    -d "login_hint=lti-learner-42" \
    -d "target_link_uri={{ request.host_url.rstrip('/') }}/learn/some-slug"
{
  "lti_version": "{{ lti_version }}",
  "message_type": "LtiResourceLinkRequest",
  "callback_received": true
}

Step 3 — Deep linking + grade passback

After launch, instructors can use Coursera's deep-linking picker to embed a specific course / module. Grades flow back via the LTI Assignment and Grade Services 2.0 spec to the LMS gradebook.

Status

LTI {{ lti_version }} integrations are certified against IMS Global's reference test suite. Production traffic is monitored on the uptime and healthz endpoints.
