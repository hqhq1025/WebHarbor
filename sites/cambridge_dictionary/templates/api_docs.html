Cambridge Dictionary — Public API v1

The Cambridge Dictionary mirror exposes a small read-only JSON API for third-party integrations. All endpoints are unauthenticated and rate- limited at the reverse proxy.

GET /api/v1/word/<slug>

Returns a JSON document describing a single dictionary entry. Example request: 

GET /api/v1/word/serendipity HTTP/1.1
Host: dictionary.cambridge.org
Accept: application/json

Example response:

{
  "headword": "serendipity",
  "slug": "serendipity",
  "pos": "noun",
  "level": "C1",
  "domain": "",
  "defined_term_id": "dt-legacy-serendipity",
  "ipa_uk": "/ˌser.ənˈdɪp.ə.ti/",
  "ipa_us": "/ˌser.ənˈdɪp.ə.ti/",
  "definitions": [ ... ]
}

The domain key is one of medical, legal, academic, business, it, or the empty string for general entries.

GET /api/suggest?q=<prefix>

Low-latency autosuggest. Returns up to 8 headword matches plus a latency_ms measurement. Documented performance target: < 50 ms on the production catalog. 

{
  "q": "ser",
  "suggestions": [
    {"headword": "serendipity", "slug": "serendipity", "level": "C1"},
    ...
  ],
  "latency_ms": 4.3,
  "target_ms": 50,
  "within_target": true
}

GET /sitemap.xml & /sitemap/letter/<a-z>.xml

Per-letter paginated sitemaps; the root sitemap is a sitemap-index that references each per-letter file. Used by search engines for incremental discovery.

GET /robots.txt

Allow-by-default policy. /api/save-word and /account are disallowed; the file also points crawlers at the sitemap.
