Frequently asked questions

How does GitHub Advanced Security pricing work?

Advanced Security is included with GitHub Enterprise Cloud and Enterprise Server, billed per active committer across the organization. Public repositories receive code scanning, secret scanning, and Dependabot at no cost. See full pricing →

How does CodeQL differ from traditional SAST tools?

CodeQL treats source code as queryable data. Instead of pattern-matching strings, it builds a semantic database of every variable, call site, and data-flow path, then runs declarative queries that ask questions like "is any tainted input ever passed to a SQL function?" That's why CodeQL catches whole-program bugs that line-by-line scanners miss.

What languages does CodeQL support?

C, C++, C#, Go, Java, JavaScript, TypeScript, Kotlin, Python, Ruby, and Swift. Coverage for additional languages ships every quarter — see the language matrix.

How fast does secret scanning detect a leaked credential?

Push events are scanned within seconds. With push protection enabled, secrets that match a partner-defined pattern are blocked at git push time before they ever land on GitHub's servers — the developer is shown an inline warning and given the option to remediate.

Can I use Advanced Security alongside my existing security tooling?

Yes. SARIF upload is supported, so any third-party SAST tool can write its findings directly into the GitHub code-scanning UI. CodeQL also exposes a CLI you can wire into existing pipelines on Jenkins, GitLab CI, or self-hosted runners.

Is GitHub Advanced Security FedRAMP / SOC 2 compliant?

GitHub Enterprise Cloud holds SOC 1 Type 2, SOC 2 Type 2, ISO 27001:2013, and is in the FedRAMP Moderate "In Process" stage. Compliance attestations are available to Enterprise customers in the Security Center.
