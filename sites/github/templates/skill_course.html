Prerequisites

This is a beginner-friendly course. You'll get the most out of it if you're already comfortable with the basics covered in Introduction to GitHub — creating a repository, opening a pull request, and merging branches. No local Git client is required; everything happens in the GitHub web editor.

  • A free GitHub account
  • A modern web browser (Chrome, Firefox, Safari, or Edge)
  • About {{ course.duration }} of focused time

Course modules

Each module is graded automatically by the Skills bot the moment you push your changes. Modules build on each other — finish them in order.

1

Set up the practice repository

Use the course template to create a fresh repository in your own account. The Skills bot opens the first issue and walks you through the workflow you'll follow for the rest of the course.

2

Trigger your first conflict

Open two branches that touch the same line of the same file. Open a pull request and watch GitHub mark it as "this branch has conflicts that must be resolved."

3

Resolve the conflict in the web editor

Use the in-browser merge editor to choose between incoming and current changes — or to keep both. Commit the resolution and watch the conflict marker disappear.

4

Mark as resolved & complete the merge

Click "Mark as resolved", complete the merge, and confirm the pull request closes cleanly. The Skills bot verifies your final state and grades the module.

5

Practice on a harder conflict

Repeat the workflow on a more realistic conflict that touches three files at once — including a binary file that requires a different resolution strategy.

6

Review & earn your badge

The bot opens a final review pull request, adds your completion badge to your profile, and links you to the next recommended course in the path.
