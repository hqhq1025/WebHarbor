Partner API v3 — Developer Documentation

Build integrations against the Booking platform. v3 supports OAuth 2.0 client-credentials and authorization-code grants, signed webhooks, and JSON-RPC bookings calls.

OAuth 2.0

The base path for OAuth is /partner/api/v3/oauth. Both authorize and token endpoints accept form-encoded and JSON bodies.

Authorize

GET /partner/api/v3/oauth/authorize?client_id=demo&redirect_uri=https://example.com/callback

Returns a JSON object with authorize_url, response_type_supported, code, redirect_uri, and scope.

Token

POST /partner/api/v3/oauth/token
client_id=demo&client_secret=demo&grant_type=client_credentials

Returns a JSON object with access_token (prefix bkp_v3_), token_type = Bearer, expires_in = 3600, and the granted scope.

Supported grant types

  • client_credentials
  • authorization_code
  • refresh_token

Supported response types

  • code
  • token

Webhooks

See /webhook/new-booking for the documented event-type, retry policy, and signature scheme.

Rate limits

Default partner quota: 600 requests / minute / token. Burst capacity: 1000 requests. Contact partner support to raise.
